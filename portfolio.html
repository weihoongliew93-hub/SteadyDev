POS & Performance

Offline POS Sync Optimization & Staging DB Innovation

.NET Multi-threading Entity Framework Azure Blob Storage

Offline POS sync was painfully slow — tables downloaded one-by-one, testing polluted production with fake data, and image downloads crawled through a web-service middleman. Introduced a staging local database by dynamically cloning tables through the existing EF middleware, added multi-threading for parallel downloads, and pointed image downloads directly at blob storage. Sync time is now set by the single largest table instead of cumulative sequential processing.
